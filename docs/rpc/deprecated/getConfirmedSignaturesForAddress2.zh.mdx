返回包含给定地址在其`accountKeys`列表中的确认交易的签名。从提供的签名或最近的确认区块开始，向后返回签名。

<Callout type={"warning"} title={"已弃用方法"}>
  预计此方法将在 `solana-core` v2.0 中移除。请改用
  [getSignaturesForAddress](/docs/rpc/http/getSignaturesForAddress)。
</Callout>

<DocSideBySide>

<DocLeftSide>

### 参数

- **account address** (`string`, required): 作为base-58编码的字符串的账户地址
- **配置对象** (`object`, optional): 包含以下字段的配置对象:

<Field
  name="commitment"
  type="string"
  defaultValue="finalized"
  optional={true}
  href="/docs/rpc/index.mdx#configuring-state-commitment"
></Field>

<Field name="limit" type="number" optional={true}>
  要返回的最大交易签名数（介于1和1,000之间，默认为1,000）。
</Field>

<Field name="before" type="string" optional={true}>
  从该交易签名开始向后搜索。（如果未提供，则搜索从最高最大确认区块的顶部开始。）
</Field>

<Field name="until" type="string" optional={true}>
  在达到限制之前找到该交易签名时，搜索直到此交易签名。
</Field>

</DocLeftSide>

<DocRightSide>

### 代码示例

```bash
curl http://localhost:8899 -X POST -H "Content-Type: application/json" -d '
  {
    "jsonrpc": "2.0",
    "id": 1,
    "method": "getConfirmedSignaturesForAddress2",
    "params": [
      "Vote111111111111111111111111111111111111111",
      {
        "limit": 1
      }
    ]
  }
'
```
