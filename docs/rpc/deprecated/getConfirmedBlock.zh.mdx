返回与提供的 Pubkey 相关联的账户的所有信息

### 参数

- **Pubkey** (`string`, required): 要查询的账户的公钥，以 base-58 编码的字符串形式
- **配置对象** (`object`, optional):
  - `commitment` (`string`, optional): [配置状态承诺](/docs/rpc/index.mdx#configuring-state-commitment)
  - `encoding` (`string`, optional): [解析响应](/docs/rpc/index.mdx#parsed-responses)的编码格式
    - `base58`: 缓慢且仅限于少于 129 字节的账户数据
    - `base64`: 返回任何大小账户数据的 base64 编码数据
    - `base64+zstd`: 使用 [Zstandard](https://facebook.github.io/zstd/) 压缩账户数据并进行 base64 编码
    - `jsonParsed`: 尝试使用程序特定的状态解析器返回更易读和明确的账户状态数据

    <details>
    <summary>更多详情</summary>

    - 当请求 `jsonParsed` 但找不到解析器时，字段会回退到 `base64` 编码，当 `data` 字段类型为 `string` 时可检测到
    </details>

- **dataSlice** (`object`, optional): 请求账户数据的一个片段
  - `length` (`usize`): 要返回的字节数
  - `offset` (`usize`): 开始读取的字节偏移量
  <Callout type={"info"}>
  仅支持 `base58`、`base64` 或 `base64+zstd` 编码的数据切片。
  </Callout>
- **minContextSlot** (`number`, optional): 请求可以在其中评估的最小槽

### 结果

结果将是一个 RpcResponse JSON 对象，其 `value` 等于：

- `<null>`: 如果请求的账户不存在
- `<object>`: 否则，一个包含以下内容的 JSON 对象:
  - `lamports` (`u64`): 分配给该账户的拉姆波数量，作为 u64
  - `owner` (`string`): 该账户分配给的程序的 base-58 编码的公钥
  - `data` (`[string, encoding]|object`): 与该账户相关联的数据，可以是编码的二进制数据或 JSON 格式 `{<program>: <state>}` - 取决于编码参数
  - `executable` (`bool`): 布尔值，指示账户是否包含程序（且严格为只读）
  - `rentEpoch` (`u64`): 下次该账户将欠租的时期，作为 u64
  - `size` (`u64`): 账户的数据大小

### 代码示例

```bash
curl http://localhost:8899 -X POST -H "Content-Type: application/json" -d '
  {
    "jsonrpc": "2.0",
    "id": 1,
    "method": "getAccountInfo",
    "params": [
      "vines1vzrYbzLMRdu58ou5XTby4qAqVRLmqo36NKPTg",
      {
        "encoding": "base58"
      }
    ]
  }
'
